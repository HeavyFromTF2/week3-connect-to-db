Stop:


## ⚔️ AI vs Me (Stage 6 CONTAINER)

### Original Prompt Used
> "Create a Node.js and Express REST API for a todo list connected to PostgreSQL (with every CRUD operation), with swagger and express. Upon startup, create the tasks table if it doesn't exist and populate it with three initial tasks only if it is empty. Create a Dockerfile and a compose.yaml file the respective services and I want to run everything using a single docker compose up command, with a env file, and use queries safe from SQL injection (like $1). send me the fileszipped and ready to be ran, on the 3001 port."

### Analysis & Differences Found

**1. What AI did better:**
- **Infrastructure Resilience:** The AI correctly implemented a `healthcheck` using `pg_isready` in `compose.yaml`. It also combined this with a custom `waitForDb()` function in the Node.js code that retries the database connection with exponential backoff. This ensures the API never crashes if Postgres takes a few extra seconds to boot.
- **Docker Image Optimization:** It used `node:20-alpine` as the base image and included the `--omit=dev` flag in the `npm install` step, which creates a much smaller and production-ready container.
- **Clean Swagger Architecture:** Instead of cluttering the main server file, it separated the OpenAPI configuration into a dedicated `config/swagger.js` file and used JSDoc comments directly above the routes. The entire structure is way more clean and production ready.

**2. What AI got wrong or ignored (Breaking the A1/A2 requirements):**
- **Schema Changes:** The AI changed the boolean column name from `done` to `completed` (even tho it was not specified in my prompt) and added unnecessary columns like `description`, `created_at`, and `updated_at`. 
- **Endpoint Paths:** It prefixed all routes with `/api/tasks` instead of the original `/tasks` root path.
- **Status Codes:** For the `DELETE` operation, the assignment expects a `204 No Content`. The AI returned a `200 OK` with a JSON success message.

**3. What my prompt forgot to specify:**
- I forgot to specify the exact table schema (`id`, `title`, `done`). Because of this, the AI hallucinated a more complex schema.
- I forgot to specify the exact base URL path (e.g., "Use `/tasks` as the base route").
- I didn't enforce the specific HTTP status codes required by the assignment (200, 201, 204, 400, 404), leaving the AI to guess standard REST conventions.

### Rematch
*After updating the prompt to explicitly define the table schema (`id`, `title`, `done`), force the `/tasks` route, and specify exact HTTP status codes (especially `204` for DELETE), the regenerated code perfectly matched the manual Stage 1-5 requirements while keeping the robust Docker setup.*
