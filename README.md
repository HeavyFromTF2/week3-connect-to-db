# Todo List API - FlyRank Week 2 Assignment

A simple CRUD API built with Node.js and Express for FlyRank Week 2.

## How to Run

1. Install dependencies:
   npm install

2. Start the server:
   node index.js

The server will run on http://localhost:3000.

Just clone this repo and try it out.

## API Documentation

You can test all endpoints visually using Swagger UI:
* Swagger UI: http://localhost:3000/docs

## Available Endpoints

* GET / - API Info
* GET /health - Health Check
* GET /tasks - List all tasks (supports filtering via `?done=true` or `?done=false`)
* GET /tasks/:id - Get a task by ID
* POST /tasks - Create a task
* PUT /tasks/:id - Update a task
* DELETE /tasks/:id - Delete a task

## AI vs Me (Stage 7 - Rematch)

### 1. The Prompt I Used
"make a simple node.js project that includes a simples todo list api for every CRUD operation, considering it must have a swagger UI dashboard in the end and that it must use express. Make it the most simple possible on a single file"

### 2. What did the AI do better?
* **Comments and structure:** The code generated by the AI in `server.js` is very clean, organized, and has descriptive comments for every single part, which makes it easy to read.
* **No extra files:** It integrated Swagger directly into the javascript code using JSDoc comments, avoiding the need to manage a separate JSON file.

### 3. What did it get wrong or ignore from my prompt?
* **Route Names:** Even though I asked for a "todo list api", it named all the endpoints `/todos` instead of `/tasks` (which is what my assignment required).
* **Missing Things:** It quietly decided to run on port 3000 and redirect `/` to the documentation, entirely ignoring the `/health` check and the `?done=true` filter I manually built.

### 4. What my prompt forgot to specify:
I did not specify the exact port, the names of the routes (`/tasks`), or specific endpoints like `/health`. The AI had to guess and make those choices for me.

### 5. One Rematch Change
By changing the prompt to specify the exact port, endpoint paths (/tasks instead of /todos), and the exact list of CRUD methods, the newly regenerated AI code correctly matched my project rules without inventing paths or skipping the health check.
